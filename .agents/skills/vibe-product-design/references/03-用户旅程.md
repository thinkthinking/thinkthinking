# 第 3 章 · 用户旅程

> **产出文件：** `03-用户旅程.md`
> **输入：** 第 01 章（目标用户、痛点、场景）。
> **作用：** 从用户视角描绘"用户怎么一步步用上并爱上这个产品"。它识别出的**触点**和**啊哈时刻**，
> 是第 05 章功能设计的直接依据——先有体验，再有功能。

## 本章目标

画出目标用户从认识产品到长期使用的完整旅程，标出每一步的行为、情绪、痛点和机会点，特别是那个让用户
"哇"的**啊哈时刻**。

## 写作要求

- **基于第 01 章的真实用户和痛点**，不要凭空设计理想用户。
- **必须标出啊哈时刻**：用户第一次切身感到价值的那一刻在哪一步、多快到达、强不强。**如果旅程里找不到
  啊哈时刻，这就是产品头号缺口，直接写出来。**
- **情绪低点 = 设计优先项**：旅程里情绪最低的几步要重点标注，它们是后续功能要优先改善的地方。
- 没有真实数据支撑的情绪打分标 `[假设]`。

## 输出结构

```markdown
# 第 3 章 · 用户旅程

## 3.1 旅程图
（Mermaid journey，见 输出格式规范.md，在图上标注啊哈时刻）

## 3.2 分阶段旅程分析
按 意识 → 考虑 → 决策 → 首次体验 → 留存/推荐 五个阶段，每阶段：

### [阶段名]
- **用户目标：** 这一步用户想达成什么
- **用户行为：** 具体动作
- **接触点：** 通过什么渠道/界面与产品互动
- **情绪：** 高/中/低 + 为什么
- **痛点：** 这一步卡在哪、为什么
- **机会点：** 怎么改善（指向后续功能）

## 3.3 啊哈时刻
- **在哪一步发生：** …
- **用户体验到什么：** …
- **如何让它来得更快、更强：** …

## 3.4 关键体验指标
- 各阶段的转化/留存关键指标，以及目标值。
```
