# 风格：苹果 (Apple) Human Interface Guidelines

高保真 UI 设计稿（非线框图、非灰框占位），锚定苹果 Human Interface Guidelines 的设计语言：清晰
（Clarity）、遵从内容（Deference）、有层次感（Depth）——界面干净精致、内容优先，视觉元素服务于
信息而不是抢戏。配色用纯净的白或极浅灰背景（#FFFFFF / #F2F2F7），近黑正文（约 #1C1C1E），次要
信息用系统灰（约 #8E8E93），可交互元素统一用系统蓝（#007AFF）标示，成功/警示可少量使用系统绿
（#34C759）/ 系统红（#FF3B30）；规避高饱和撞色、炫彩渐变与重装饰。字体用类 SF Pro / 苹方的现代
无衬线：大标题粗壮醒目、正文常规字重，字号层级分明（大标题 / 标题 / 正文 / 脚注），依赖字重与
字号而非颜色区分层级。布局用分组卡片与分组列表（圆角约 10–14px 的浅色卡片浮于浅灰底之上）、
舒适的内边距与呼吸感，对齐严整、遵循清晰的栅格；关键操作用饱满的圆角按钮（系统蓝底白字）或
系统蓝文字按钮。组件质感干净利落：极浅的弥散阴影或无阴影、细分隔线、SF Symbols 一类的细线图标、
必要处可用轻微毛玻璃质感（导航栏/浮层），无拟物、无插画堆砌。整体氛围精致、可信赖、现代而克制。

界面必须使用真实文案与内容（真实标题、按钮文字、数据、列表项），不要占位框、假字或乱码；把这个
界面最关键的设计决策清楚展现出来——核心组件的形态、信息层级、以及此刻所处的交互状态。同一功能的
不同状态（默认 / 选中 / 空 / 加载 / 异常）各出一张图，用状态差异体现设计，不要把多个状态挤进一张图。
